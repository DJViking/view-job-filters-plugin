Es können so viele Fiter für Reguläre Ausdrücke wie gewünscht verwendet werden. Die Filter werden in der Reihenfolge angewendet in der sie angeordnet sind.

Wert: Es kann aus den folgenden Werten gewählt werden:
  • Job: Name
  • Job: Vollständiger Name: Vollständiger Name inklusive Name des Ordners, ohne führendes '/'
  • Job: Anzeigename
  • Ordner: Vollständiger Name: Vollständiger Pfad aller Ordner, z.B. 'topfolder/subfolder'
  • Job-Beschreibung
  • SCM-Konfiguration: Auswahl anhand des SCMs ist nützlich wenn man hunderte von Jobs hat, die automatisch organisiert werden sollen, und die im SCM bereits organisiert sind. (Zieht momentan die Pfade/Konfiguration für CVS, SVN und git heran.)
  • Email-Empfänger: Auswahl anhand der Email ist ein nützliches Kriterium für "Wer interessiert sich für diesen Job?"
  • Maven-Konfiguration
  • Build-Zeitplan: Vergleich mit dem CRON Ausdruck (und Kommentaren) für zeitgesteuerte Builds und Builds anhand regelmäßiger SCM-Abfragen
  • Node Label Ausdruck: Vergleich mit dem Label-Ausdruck der "Beschränke, wo dieses Projekt ausgeführt werden darf" Option.

Filter-Typ: Ob dieser Filter zusätzliche Jobs hinzufügt oder bestehende herausfiltert.
