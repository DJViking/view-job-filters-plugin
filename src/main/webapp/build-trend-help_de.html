Dieser Filter inkludiert/exkludiert Jobs abhängig davon wie Builds gestart wurden und geendet haben.
Um alle Builds zu überprüfen, kann die Option "Zurückliegender Bereich" entweder auf 0 gesetzt oder leer gelassen werden.

Beachte, dass nicht alle Kombinationen von Kriterien sinnvoll sein müssen. Falls dieser Filter komplizierter als nötig erscheint, könnten der "Build-Status Filter" und der "Job-Status Filter" hilfreich sein.
This filter includes/excludes jobs based on how their builds were started and how they finished.

Options

Build-Trend:
Gibt an wie die Builds eines Jobs gestart wurden oder geendet haben müssen, damit dieser Filter auf einen Job zutrifft.
Zurückliegender Bereich:
Ein Job kann viele Builds haben und diese Option bestimmt, wie viele dieser Builds zur Überprüfung herangezogen werden. Die Angabe geht rückwärts vom letzten Build aus und kann eine Dezimalzahl sein, z.B. 1.5 Stunden. Um alle Builds heranzuziehen kann 0 als Wert angegeben werden.
Filter-Typ:
Gibt an ob Jobs hinzugefügt oder entfernt werden sollen. Es gibt zwei mögliche Arten von Filter-Typen:
  • Zutreffende/Unzutreffende aufnehmen: fügt zusätzliche Job zu dieser Ansicht hinzu.
  • Zutreffende/Unzutreffende auschließen: entfernt Jobs aus dieser Ansicht, die durch vorherige Filter hinzugefügt wurden.

Beispiele

  • Alle Jobs, die innerhalb der letzten 3 Stunden liefen.
  • Alle Jobs, die die letzten 10 Builds stabil geblieben sind.
  • Alle Jobs, die mindestens einen nicht stabilen Build innerhalb der letzten Woche hatten.
  • Alle Jobs, die in den letzten 7 Tagen nicht ausgeführt wurden.
  • Alle Jobs, die in den letzten 30 Tagen nicht von einer SCM-Änderung ausgelöst wurden.
  • Alle Jobs, die zuletzt von einem entfernten Rechner ausgelöst wurden.
